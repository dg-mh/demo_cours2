git restore

Restaure les fichiers d'un arbre de travail

git restore --staged <fichier>

Retire un fichier de l'espace de staging de git

git restore <fichier>

Annule les modifications dans un fichier non stagé

git reset

Réinitialise votre espace de travail à un état spécifique

git reset <fichier>

Retire un fichier de l'espace de staging de git

git reset [--soft | --mixed | --hard] <commit>

Réinitialise votre branche courante sur le commit spécifié. Le mode choisis déterminera ce qu'il advient des modifications sur vos fichiers. Si aucun mode n'est choisi ce sera la mode mixed qui sera utilisé

Mode:

  • --soft: ne touche pas a vos fichiers, ce qui était stagé le reste
  • --mixed: ne touche pas à vos fichier mais réinitialise l'index; l'espace de staging sera donc vidé
  • --hard: Réinitialise vos fichier à l'état du commit cible. Les modifications sur vos fichiers seront donc perdus

Ici, on peut indiquer les commits cible de différentes façons:

  • Nom de branche
  • Nom de tag
  • Checksum d'un commit (SHA)
