git tag

git tag nous permet de créer un tag (étiquette) sur un commit précis. Les tags nous permettent de retrouver facilement un commit. Un exemple fréquent d'utilisation de tag dans un dépot de code est de spécifier différentes version de notre application.

git tag <nom_du_tag> [<commit>]

Crée un tag sur le commit pointé par HEAD ou sur un autre commit si vous en spécifiez un.

Ici, on peut indiquer le commit cible de différentes façons:

  • Nom de branche
  • Nom de tag
  • Checksum d'un commit (SHA)

git tag -d <nom_du_tag>

Permet de supprimmer un tag
